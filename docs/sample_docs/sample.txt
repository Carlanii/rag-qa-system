RAG（Retrieval-Augmented Generation）是一种结合检索与生成的 AI 技术架构。
它的核心思想是：在让大语言模型回答问题之前，先从知识库中检索出相关文档，
然后将检索到的内容作为上下文提供给模型，让模型基于真实信息生成回答。

RAG 系统的优势包括：
1. 解决大模型知识更新的问题，不需要重新训练模型
2. 减少模型幻觉，让回答有据可依
3. 可以接入私有知识库，保护数据安全

RAG 系统的典型工作流程：
1. 文档预处理：将文档分割成小段文本块
2. 向量化：用 Embedding 模型将文本块转为向量
3. 索引存储：将向量存入向量数据库（如 ChromaDB）
4. 检索：用户提问时，计算问题向量与文档向量的相似度
5. 生成：将检索到的文档块和问题一起发给 LLM 生成回答

LangChain 是一个用于构建 LLM 应用的框架，它提供了丰富的组件：
- Document Loaders：加载不同格式的文档
- Text Splitters：文档分割策略
- Vector Stores：向量数据库接口
- Chains：构建处理链
- Agents：智能体编排
